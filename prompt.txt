You are a senior full-stack architect. Scaffold a production-ready admin application with the following exact tech stack and requirements.

## Project Overview
Admin application for a mobile LMS platform called TBT (Tamil Business Tribe).
The platform has 4 core modules: LMS (courses, curriculum), Task Management, Community, and Webinar Management.
Expected concurrent users: 500 members.

---

## Tech Stack

### Admin Panel (Web)
- Framework: Next.js 14+ with App Router
- Language: TypeScript (strict mode)
- Styling: Tailwind CSS
- UI Components: shadcn/ui
- State Management: Zustand
- Data Fetching: TanStack Query (React Query)
- Form Handling: React Hook Form + Zod validation
- Auth: Clerk
- Hosting: Vercel

### Backend API
- Runtime: Node.js
- Framework: Fastify with TypeScript
- ORM: Prisma
- Validation: Zod
- Auth Middleware: Clerk JWT verification
- WebSocket: Socket.io (for real-time features)
- Job Queue: BullMQ + Redis
- Hosting: Railway

### Database & Services
- Primary Database: PostgreSQL via Supabase
- Cache: Redis via Upstash
- Media Storage: Cloudflare R2 (images, documents)
- Video Storage: Bunny Stream (course videos)
- Real-time: Supabase Realtime + Socket.io
- Push Notifications: Firebase Cloud Messaging
- Email: Resend
- SMS/OTP: Twilio
- Webinar: Agora.io
- Error Tracking: Sentry
- Monitoring: Better Stack

## Database Schema (Prisma)
Generate complete Prisma schema for:

1. Admin Table
   - id, adminId (TBT-ADMIN-2026-XXXX), fullName, email, contactNumber,
     alternateContact, dateOfBirth, bloodGroup, profilePhoto
   - role (SuperAdmin, Manager, Moderator, Viewer)
   - department, designation, reportingManagerId, employeeId
   - country, state, city, address
   - username, accountStatus (Active, Inactive, Suspended)
   - twoFactorEnabled, createdBy, notes, tags
   - timestamps

2. Member Table
   - id, memberId, fullName, email, phone
   - enrolledCourses, completedCourses, activeTasks
   - membershipStatus, joinedAt, lastActive
   - timestamps

3. Course Table
   - id, courseId, title, description, thumbnail, category
   - curriculum (JSON), totalDuration, totalLessons
   - instructor, status (Draft, Published, Archived)
   - enrollmentCount, completionRate
   - timestamps

4. Task Table
   - id, taskId, title, description, priority (Low, Medium, High, Critical)
   - status (Todo, InProgress, Review, Done)
   - assignedTo, assignedBy, dueDate, completedAt
   - attachments, tags
   - timestamps

5. Community Table
   - id, postId, authorId, content, mediaUrls
   - type (Post, Announcement, Poll)
   - likes, comments, shares, isPinned
   - timestamps

6. Webinar Table
   - id, webinarId, title, description, thumbnail
   - hostId, scheduledAt, duration, status (Scheduled, Live, Ended)
   - agoraChannelId, registeredCount, attendedCount
   - recordingUrl, timestamps

7. Notification Table
   - id, userId, title, body, type, isRead
   - metadata (JSON), timestamps

8. RBAC Table
   - id, adminId, module, canView, canCreate, canEdit, canDelete
   - timestamps
