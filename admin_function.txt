Display it in the readonly Admin ID field. The sequence must be generated server-side using a Supabase sequence or auto-increment counter. Never generate it on the client.

Backend: Create the Fastify route `GET /api/admins/generate-id` that queries the DB for the next ID, formats it, and returns it. Use a dedicated `admin_id_sequence` table or PostgreSQL sequence object to ensure no duplicates even under concurrent requests.

## Backend Routes to Generate (Fastify + TypeScript)

Generate complete, production-ready Fastify route handlers for:

1. `GET  /api/admins/generate-id` — generate next Admin ID
2. `GET  /api/admins/check-username?username=xxx` — check uniqueness
3. `GET  /api/admins/check-email?email=xxx` — check uniqueness
4. `GET  /api/admins/search?q=xxx&limit=8` — manager search
5. `POST /api/upload/presigned-url` — R2 presigned upload URL
6. `GET  /api/auth/generate-password` — secure password generation
7. `GET  /api/location/states?country=xxx` — state list
8. `GET  /api/location/cities?state=xxx` — city list
9. `POST /api/admins` — create admin (full logic below)

For `POST /api/admins`:
- Validate entire payload with Zod
- Check email uniqueness
- Check username uniqueness
- Hash password using bcrypt (salt rounds: 12)
- Upload profilePhoto URL is already done client-side — just store the URL
- Insert admin record into PostgreSQL via Prisma
- Insert RBAC records into rbac table (one row per module)
- Send welcome email via Resend with username + temporary password
- Send welcome SMS via Twilio with login URL
- Create audit log entry (who created, when, IP address)
- Return created admin object (exclude password hash)
- All DB operations in a Prisma transaction — if any step fails, rollback everything

## Error Handling Requirements

Every API route must:
- Return consistent error format: `{ success: false, error: { code: string, message: string, fields?: Record<string, string> } }`
- Return consistent success format: `{ success: true, data: T }`
- Log errors to Sentry with full context (admin ID, route, payload shape without sensitive data)
- Never expose stack traces or internal error details to the client
- Rate limit the create admin endpoint: max 10 requests per minute per IP
